# Contributing to DevDen

DevDen is a structured and public repository representing SIG AI members and their technical work. All contributions must follow the standards below to maintain consistency, professionalism, and long-term scalability.

---

## Contribution Workflow

1. Clone the repository.
2. Create a feature branch.

Branch naming format:

`feature/<batch>/<FullName>`

Example:

`feature/2024-2028/JohnDoe`

3. Make changes only within your designated folder.
4. Open a Pull Request for review.

Direct pushes to `main` are not permitted.

---

## Repository Structure Rules

Members may:

- Create and update only their own folder within their respective batch directory.
- Add project folders inside their personal directory.
- Update their own README and project documentation.

Members may not:

- Modify another member’s folder.
- Rename batch directories.
- Alter root-level structure.
- Modify batch README files (maintainer responsibility).
- Make structural changes without approval.

---

## Documentation Standards

All documentation must follow the templates provided in the `docs/` directory:

- `docs/MEMBER_TEMPLATE.md`
- `docs/PROJECT_TEMPLATE.md`

Batch README updates must follow:

- `docs/BATCH_TEMPLATE.md` (Maintainers only)

Each member folder must contain:

- A properly structured `README.md`
- Organized project directories

Each project folder must contain:

- Complete source code
- A project-level `README.md`

---

## Pull Request Requirements

Every Pull Request must:

- Follow the defined folder structure
- Use clear and descriptive commit messages
- Ensure all deployment links are functional
- Avoid unnecessary or unrelated files

At least one approval is required before merging.

---

## Professional Conduct

DevDen is publicly visible and powers the official DevDen website. All contributions must reflect professionalism, clarity, and technical quality.

By contributing, you agree to adhere to these standards.
